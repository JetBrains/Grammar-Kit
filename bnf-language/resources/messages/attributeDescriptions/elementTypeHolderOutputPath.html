A path to generate the IElement-type holder class in, resolved relative to the BNF file's parent directory. When unset, the holder is written next to the parser (in pure-Java mode) or next to the PSI files (in syntax-API/Kotlin mode), preserving previous behavior.

Examples:


  {
    elementTypeHolderClass="com.example.MyTypes"
    //Given that BNF file is located at "path/to/project/grammar.bnf"
    //Resulting holder file would be placed in "path/to/project/generated/types/com/example/MyTypes.java"
    elementTypeHolderOutputPath="generated/types"
  }

CLI override: the headless generator accepts --elementTypeHolderOutputPath <path>. When both this attribute and the CLI flag specify different paths, the CLI value wins and a warning is printed. Pass --strict-paths to make such conflicts fatal.
