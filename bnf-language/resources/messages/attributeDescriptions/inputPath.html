A default source root to look up FQN-valued input attributes (such as parserUtilClass, psiImplUtilClass, mixin, extends, implements), resolved relative to the BNF file's parent directory. When set, narrows the IDE's PsiHelper class lookups from the project-wide scope to the named directory. Has no effect on headless / classloader-based resolution (AsmHelper, ReflectionHelper).

For PSI-side overrides (psiImplUtilClass, per-rule mixin and implements) see psiInputPath.

Examples:


  {
    //Given that BNF file is located at "path/to/project/someModule/src/main/grammars/grammar.bnf"
    //All FQN inputs are resolved within "path/to/project/someModule/src/main/java"
    inputPath="../java"
  }

CLI override: the headless generator accepts --input-path <path>. When both this attribute and the CLI flag specify different paths, the CLI value wins and a warning is printed. Pass --strict-paths to make such conflicts fatal.
