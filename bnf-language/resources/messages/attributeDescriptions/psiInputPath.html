The source root that holds the user-authored PSI-side classes referenced by psiImplUtilClass, per-rule mixin, and per-rule implements, resolved relative to the BNF file's parent directory. When set, narrows the IDE's PsiHelper.findClass scope for those three attributes to the named directory. Falls back to the global inputPath default when unset, and finally to project-wide search. In the headless generator the value is consulted only when --source-psi is passed: it then joins inputPath as a source root for JavaSyntaxHelper (with AsmHelper as fallback). Otherwise headless / classloader-based resolution (AsmHelper, ReflectionHelper) ignores the attribute.

One knob covers all three because psiImplUtilClass, mixin targets, and implements interfaces typically live together in the same source root.

Examples:


  {
    //BNF at "path/to/project/grammars/grammar.bnf" -> mixin / implements / psiImplUtilClass
    //classes resolved against "path/to/project/grammars/../psi-src"
    psiInputPath="../psi-src"
    psiImplUtilClass="com.example.MyPsiImplUtil"
  }
  myRule ::= ... { mixin="com.example.MyRuleMixin" implements="com.example.MyMarker" }

CLI override: the headless generator accepts --psi-input <path>. When both this attribute and the CLI flag specify different paths, the CLI value wins and a warning is printed. Pass --strict-paths to make such conflicts fatal.
