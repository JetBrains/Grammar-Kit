A path to generate the Kotlin syntax-element-type holder in, resolved relative to the BNF file's parent directory. Applies only when the syntax-API parser is enabled (generate=[parser-api="syntax"]).

Examples:


  {
    generate=[parser-api="syntax"]
    syntaxElementTypeHolderClass="com.example.MySyntaxTypes"
    //Given that BNF file is located at "path/to/project/grammar.bnf"
    //Resulting holder file would be placed in "path/to/project/generated/syntax-types/com/example/MySyntaxTypes.kt"
    syntaxElementTypeHolderOutputPath="generated/syntax-types"
  }

CLI override: the headless generator accepts --syntaxElementTypeHolderOutputPath <path>. When both this attribute and the CLI flag specify different paths, the CLI value wins and a warning is printed. Pass --strict-paths to make such conflicts fatal.
